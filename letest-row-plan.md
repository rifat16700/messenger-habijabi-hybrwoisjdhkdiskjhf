আল্টিমেট প্রজেক্ট ব্রিফ: সার্ভারলেস ও ডেটাবেস-ফ্রি পিটুপি চ্যাট অ্যাপ (The Hybrid Engine)

এটি একটি ক্রস-প্ল্যাটফর্ম (Android, iOS, Web) চ্যাট এবং কলিং অ্যাপ, যার মূল লক্ষ্য হলো জিরো-সার্ভার-কস্ট এবং ম্যাক্সিমাম প্রাইভেসি।

নিচে এই অ্যাপের প্রতিটি প্রযুক্তির পরিচয়, ব্যবহারের কারণ এবং কাজের ধরন ধাপে ধাপে দেওয়া হলো:

১. ফ্রেমওয়ার্ক: React Native + Expo

কী (Intro): এটি অ্যাপের ফ্রন্টএন্ড বা ইউজার ইন্টারফেস (UI) বানানোর টুল।

কেন ব্যবহার করব (Why): আলাদা করে অ্যান্ড্রয়েডের জন্য Java এবং ওয়েবসাইটের জন্য HTML/CSS না লিখে, শুধু একবার কোড লিখেই মোবাইল অ্যাপ (APK) এবং ওয়েবসাইট দুটোই পাওয়ার জন্য।

কীভাবে কাজ করবে (What): Antigravity এই ফ্রেমওয়ার্কে কোড লিখবে। ইউজার চাইলে অ্যাপ ইন্সটল করে চালাতে পারবে, আবার অ্যাপ না থাকলে সরাসরি ব্রাউজার থেকে ওয়েবলিংক ব্যবহার করে কল বা চ্যাট করতে পারবে।

২. সিগন্যালিং সার্ভার: Hugging Face Space (Node.js)

কী (Intro): এটি আমাদের অ্যাপের "ট্রাফিক পুলিশ" বা রিলে ইঞ্জিন।

কেন ব্যবহার করব (Why): দুজনের মধ্যে WebRTC কল কানেক্ট করার জন্য আইপি অ্যাড্রেস আদান-প্রদান করতে এবং কে অনলাইনে আছে তা ট্র্যাক করতে।

কীভাবে কাজ করবে (What): যখন User A কল বা মেসেজ দেবে, হাগিংফেস সেটা রিসিভ করে User B-এর কাছে পাস করে দেবে। অ্যাপ ব্যাকগ্রাউন্ড থেকে দিনে ২-১ বার "Ping" পাঠাবে, যার ফলে হাগিংফেসের সার্ভার কখনোই "Sleep Mode"-এ যাবে না। সার্ভারটি কোনো ডেটা সেভ রাখবে না।

৩. অনলাইন চ্যাট ও ফাইল ট্রান্সফার: WebRTC Data Channel

কী (Intro): এটি সরাসরি এক ডিভাইস থেকে আরেক ডিভাইসে ডেটা পাঠানোর টেকনোলজি (P2P)।

কেন ব্যবহার করব (Why): ফাইলের সাইজ লিমিট (যেমন ২ জিবি বা ৫ জিবি) আনলিমিটেড করার জন্য এবং সার্ভারের ব্যান্ডউইথ খরচ জিরো করার জন্য।

কীভাবে কাজ করবে (What): দুজন ইউজার অনলাইনে থাকলে, মেসেজ বা ফাইল কোনো ডেটাবেস বা সার্ভারের ওপর দিয়ে যাবে না। সরাসরি User A-এর ফোন থেকে User B-এর ফোনে ট্রান্সফার হবে।

৪. অফলাইন ডেটা বাফার: The file.io & JSONBin Hack

কী (Intro): ইউজার অফলাইনে থাকলে ডেটা সাময়িকভাবে ধরে রাখার জাদুকরী ট্রিক।

কেন ব্যবহার করব (Why): விலையுயர்ந்த (দামি) ক্লাউড ডেটাবেস (Supabase/Firebase) ব্যবহার না করে সম্পূর্ণ ফ্রিতে অফলাইন মেসেজ বা বড় ফাইল সেভ রাখার জন্য।

কীভাবে কাজ করবে (What): User B অফলাইনে থাকলে, User A-এর মেসেজ/ফাইলগুলো হাগিংফেস সার্ভার file.io-তে আপলোড করে দেবে। file.io একটা গোপন লিংক দেবে। হাগিংফেস সেই লিংকটা JSONBin.io  একটা টেক্সট হিসেবে সেভ করে রাখবে। User B অনলাইনে আসামাত্রই লিংকটা থেকে ডেটা ডাউনলোড হবে এবং সার্ভার থেকে ফাইলটা চিরতরে মুছে যাবে (Burn after reading)।

৫. অফলাইন রিংগিং: FCM / Expo Push Notifications

কী (Intro): ইন্টারনেট চালু থাকা অবস্থায় কিন্তু অ্যাপ বন্ধ থাকলে ইউজারের ফোনে নোটিফিকেশন বা কল পৌঁছানোর সিস্টেম।

কেন ব্যবহার করব (Why): একদম মেসেঞ্জার বা টেলিগ্রামের মতো রিয়েলিস্টিক ফিল দেওয়ার জন্য, যেন অ্যাপ বন্ধ থাকলেও কল আসলে ফোনে ফুল-স্ক্রিন রিং হয়।

কীভাবে কাজ করবে (What): হাগিংফেস দেখবে User B অফলাইনে। সে গুগলের ফায়ারবেস (FCM) সার্ভারকে একটা সিগন্যাল দেবে। ফায়ারবেস User B-এর বন্ধ ফোনে একটা "Wake Up" পুশ নোটিফিকেশন পাঠাবে, যা ফোনের স্ক্রিনে "Incoming Call" ভাসিয়ে তুলবে এবং রিংটোন বাজাবে।

৬. কনফারেন্স কলিং: Tree / Sub-host Routing

কী (Intro): ৩ বা তার বেশি মানুষের গ্রুপ কলিং লজিক।

কেন ব্যবহার করব (Why): পিওর P2P (Mesh) সিস্টেমে ৫ জন একসাথে কল করলে মোবাইল হ্যাং করে এবং ইন্টারনেট ক্র্যাশ করে। এটা রোধ করার জন্য।

কীভাবে কাজ করবে (What): যে কল শুরু করবে, সে শুধু ২ জনকে তার ভিডিও পাঠাবে (Sub-hosts)। ওই ২ জন বাকিদের কাছে ভিডিও ফরোয়ার্ড করবে। কেউ কল থেকে বেরিয়ে গেলে হাগিংফেস সার্ভার সাথে সাথে বাকিদের অন্য লাইনে রি-কানেক্ট (Peer Healing) করে দেবে।

৭. গ্লোবাল ইউজার ডেটাবেস: Supabase Auth

কী (Intro): অ্যাপের একমাত্র স্থায়ী ডেটাবেস।

কেন ব্যবহার করব (Why): ইউজারদের সিকিউর লগইন, ইউজারনেম দিয়ে সার্চ করা এবং অ্যাডমিন প্যানেল থেকে কন্ট্রোল করার জন্য।

কীভাবে কাজ করবে (What): এখানে শুধুমাত্র ইউজারের নাম, ইমেইল/পাসওয়ার্ড এবং প্রোফাইল পিকচারের (ImgBB API) লিংক shoho account er bacic shobkichu থাকবে। কেউ ইউজারনেম লিখে সার্চ দিলে, এই ডেটাবেস থেকেই তাকে খুঁজে বের করে কল লাগানো হবে।

৮. অ্যাডমিন প্যানেল এবং ব্যাজ সিস্টেম (SVG)

কী (Intro): অ্যাপের কন্ট্রোল রুম।

কেন ব্যবহার করব (Why): ইউজার ম্যানেজমেন্ট এবং স্পেশাল ইউজারদের (যেমন ভেরিফাইড বা মডারেটর) আলাদা অ্যাবিলিটি দেওয়ার জন্য।

কীভাবে কাজ করবে (What): অ্যাডমিন SVG কোড দিয়ে বিভিন্ন ব্যাজ ডিজাইন করে সেভ করে রাখবে। নির্দিষ্ট ইউজারকে সেই ব্যাজ অ্যাসাইন করলে, তার প্রোফাইলে ব্যাজটি শো করবে এবং সে ওই ব্যাজের সাথে যুক্ত স্পেশাল ফিচারগুলো (যেমন: বড় ফাইল পাঠানো বা গ্রুপ খোলা) ব্যবহার করতে পারবে।

সারসংক্ষেপ: এই আর্কিটেকচারের ফলে পুরো অ্যাপটিতে সার্ভার স্টোরেজ কস্ট হবে ০ (জিরো), অথচ ইউজাররা পাবে আনলিমিটেড ফাইল ট্রান্সফার এবং হাই-প্রাইভেসি চ্যাটিং অভিজ্ঞতা!
